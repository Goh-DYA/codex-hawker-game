# Support Guide

Status: Pre-release template; monitored public contact not yet assigned

## Player self-help

If the game does not start:

1. Use a supported current desktop browser and reload once online.
2. Disable page-specific script/content blockers temporarily only if you trust the deployment origin.
3. Check that browser storage and service workers are allowed for the site.
4. Do not clear site data if you need to preserve a save.

If progress does not load, wait for any save/recovery message. Do not repeatedly reset or clear storage. Use a manual save export if the release build provides one.

If performance is low, select Lower-end quality, reduce decorative effects/audio voices, close high-load tabs, and keep browser zoom/render scale reasonable. Report the exact browser version and quality mode.

## Safe issue report

Include:

- game version/build shown in Settings;
- browser version, operating system, viewport, and quality mode;
- what you expected, what happened, and repeatable steps;
- whether you were offline, updating, or loading an older save;
- a screenshot without personal information;
- exported diagnostic/save data only if the UI clearly explains its contents.

Never include passwords, payment data, identity documents, private browsing history, or unrelated browser files. There are no official support accounts or payments in 1.0.

## Data-loss triage

Support must first preserve current and backup slots. Record whether the failure followed quota pressure, private mode, browser cleanup, update, migration, crash, or manual reset. Never instruct a player to clear site data before recovery/export options are exhausted.

## Security and cultural reports

Security reports need a private monitored channel before release; do not publish exploit details containing player data. Cultural accuracy, brand, translation, accessibility, or food-safety wording concerns should be acknowledged and routed to the appropriate human reviewer.

## Before publication

- Assign and test a monitored support contact and private security channel.
- Define response expectations and retention/deletion policy for submitted reports.
- Provide browser-specific storage/export instructions.
- Train support not to request unnecessary personal data.
- Link the final privacy notice and known customer-facing issues.
